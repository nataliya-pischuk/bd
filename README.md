# Домашнее задание к занятию «Базы данных»
## Задание 1
Опишите не менее семи таблиц, из которых состоит база данных. Определите:

какие данные хранятся в этих таблицах
Какой тип данных используется в столбцах этих таблиц, если данные хранятся в PostgreSQL?
Нарисуйте схему полученной модели данных. Можно использовать онлайн-редактор: https://app.diagrams.net/

Этапы реализации:

- Внимательно изучите предоставленный вам файл с данными и подумайте, как можно сгруппировать их по смыслу.
- Разбейте исходный файл на несколько таблиц и определите список столбцов в каждой из них.
- Для каждого столбца подберите подходящий тип данных в PostgreSQL.
- Для каждой таблицы определите первичный ключ (PRIMARY KEY).
- Определите типы связей между таблицами.
- Начертите схему модели данных. На схеме должны быть чётко обозначены:
  все таблицы с их названиями
  все столбцы с указанием типов данных
  первичные ключи (они должны быть явно выделены),
  линии, показывающие связи между таблицами.
- Результатом выполнения задания должен стать скриншот получившейся схемы базы данных.

## Решение 2

1 Таблица – Сотрудники  - employees - таблица хранит информацию о сотрудниках
employee_id Уникальный идентификатор сотрудника SERIAL PRIMARY KEY 
full_name ФИО сотрудника TEXT 
salary - размер оклада NUMERIC
device_date Дата найма DATE
position_id Внешний ключ к должности INT REFERENCES positions(position_id) 
department_id Внешний ключ к подразделению INT REFERENCES departments(department_id) 
branch_address_id Внешний ключ к адресу филиала INT REFERENCES branch_addresses(branch_address_id)

2 Таблица – Должности  - position
position_id SERIAL PRIMARY KEY
title TEXT UNIQUE
3 Таблица – Тип подразделения type_department
dep_type_id SERIAL ПЕРВИЧНЫЙ КЛЮЧ 
name TEXT УНИКАЛЬНЫЙ тип TEXT («Отдел», «Группа», «Департамент»)
4 Таблица – Структурное подразделение – structural_department
dep_id — порядковый номер, первичный ключ
name — текст
type_id - Внешний ключ к типу подразделения

5 Таблица – Адрес филиала - branch_addresses
branch_address_id SERIAL PRIMARY KEY 
address TEXT UNIQUE

6 Таблица – Проект по которому назначен – project

project_id — порядковый номер, первичный ключ, 
name — текстовое поле, уникальное

7 Таблица – employees_project – соответствие назначенного проекта сотруднику
employee_id INT Ссылается на сотрудников (employee_id) 
project_id INT Ссылается на проекты (project_id) → ПЕРВИЧНЫЙ КЛЮЧ (employee_id, project_id)
